# 个人选股 App — 需求规格说明书

> 本文档供编码 Agent 直接阅读并实现。请先读「第 1 节 设计哲学」中的两条铁律，它们是所有实现决策的约束前提，不可违背。

---

## 0. 一句话目标

构建一个**个人用、本地运行、仅面向中国 A 股（沪深京）**的选股决策支持系统：从全市场几千只 A 股中，依据可配置的规则与因子，筛选并验证出少量候选股，并附带**可热插拔的 AI 分析能力**。系统不下单、不做投资决策，只做筛选、回测与解释。

> 范围约束：本期**只做 A 股**，不含港股、美股、加密货币等其他市场。但数据源与市场相关逻辑仍以适配器/配置形式实现，便于未来扩展。

---

## 1. 设计哲学与第一性原理（必读）

选股的本质是一个**信息漏斗**：数据从上往下流，信息密度逐层提高，几千只 → 几只。整个系统就是这个漏斗。

### 铁律一：确定性防火墙（Determinism Firewall）

系统分为「确定性计算核心」与「AI 边界模块」两部分，二者严格隔离：

- **确定性核心**（技术指标计算、因子打分、条件筛选、策略回测）必须 100% 由确定性代码完成（pandas / TA-Lib / backtrader 等）。**严禁在核心内调用任何 LLM**。
- 原因：LLM 会幻觉数字、输出不可复现、且在回测中存在「前视污染」（模型训练数据已包含历史区间之后的信息，导致情绪/判断被未来知识污染，回测收益虚高）。
- AI 模块只能接在管道的**两个文本边界**：输入端（文字 → 结构化特征）与输出端（结构化数据 → 自然语言）。AI 产出必须先落地为结构化的数值/类别特征存入数据库，核心只从确定性存储读取，**运行时不得回调 AI**。

### 铁律二：AI 可热插拔（Hot-Pluggable AI）

AI 提供商、模型、API Key、以及每个 AI 功能，都必须可通过**配置文件 / 环境变量**切换或开关，**无需改动业务代码**。当 AI 被禁用或 Key 缺失时，确定性核心必须**完整可用**（优雅降级），整个选股/回测流程照常运行。详见第 4 节。

---

## 2. 系统架构（分四层）

```
数据源层  →  数据层  →  引擎层（确定性核心）  →  应用层
 几千只                                            几只
```

| 层 | 职责 | AI 是否接入 |
|---|---|---|
| 数据源层 | 行情 / 财务 / 舆情文本的外部来源 | — |
| 数据层 | 采集、清洗标准化、存储 | ✅ 接入点①（舆情文本→特征） |
| 引擎层 | 技术指标、因子打分、条件筛选、策略回测 | ❌ 禁区；仅边缘接入点②（NL→筛选条件） |
| 应用层 | 选股结果、监控预警、图表、报告/对话 | ✅ 接入点③（结果→报告/对话） |

## 3. 功能需求

### 3.1 数据层

- **数据采集（A 股）**：默认用 **akshare**（免费、无需 Key）作为主数据源，**tushare**（需注册/积分）作为备用源。数据源以适配器封装，便于切换或叠加。通过定时任务（cron / APScheduler）每个交易日收盘后（16:00 后）增量拉取。无需实时，T+1 或延迟数据即可。需覆盖：
  - **历史行情**：沪深京 A 股日/周/月线，支持不复权 / 前复权 / 后复权三种（如 `ak.stock_zh_a_hist`、`ak.stock_zh_a_daily`，部分接口可回溯至 1990 年代）。
  - **基本面**：三大财务报表与关键财务指标（如 `ak.stock_financial_*` 系列）。
  - **指数与成分**：宽基/行业指数历史行情，用于基准对比与股票池划分。
  - **股票列表与状态**：全市场代码、上市/退市/ST 状态。
- **采集健壮性**：akshare 接口可能因上游数据源改版而失效，且单次调用有数量限制——必须**分批拉取 + 本地缓存 + 增量更新**，并对接口失败做重试与告警；数据源适配器要能在主源失效时回退到备用源。
- **清洗标准化**（强制，否则上层全是假信号）：
  - 价格**前/后复权**处理（分红送股导致的价格断层）；量化回测默认用**后复权**。
  - **停牌、缺失值、涨跌停**处理；注意 A 股 T+1 与涨跌停板对回测撮合的影响。
  - **退市/ST/幸存者偏差**：回测用的股票池必须包含历史上已退市/已 ST 的标的，禁止只用当前在市标的回测。
- **存储**：默认 SQLite + Parquet（几千只 × 十年日线约数百 MB）。仅当引入分钟级数据时才允许升级到 DuckDB / 时序库。存储层需抽象，便于替换。
- AI 产出的特征（情绪分、事件标签等）作为普通列写入同一存储，供引擎层当作因子读取。

### 3.2 引擎层（确定性核心 · AI 禁区）

- **技术指标**：用 pandas-ta / TA-Lib 计算 MA、MACD、RSI、KDJ、布林带等，禁止自实现易错版本。
- **因子模型**：实现价值 / 成长 / 动量 / 质量 / 低波动等因子的打分与加权排序，因子可配置、可扩展。
- **条件筛选器**：支持多条件组合查询（如 `pe<20 & roe>15 & close>ma60`）。筛选逻辑须可命名、可保存、可复用为「策略」。
- **回测引擎**：用 backtrader / vectorbt，禁止自研以避免未来函数与前视偏差。输出收益率、年化、夏普比率、最大回撤等指标。**必须计入 A 股真实交易摩擦**：佣金、印花税（卖出单边）、过户费、滑点，并正确模拟 **T+1 交易制度**与**涨跌停无法成交**的情形。基准默认用沪深300 / 中证全指等指数。

### 3.3 应用层

- **选股结果**：候选列表，带因子评分与排序。
- **监控预警**：自选股管理 + 条件触发（价格突破、指标金叉），通过邮件 / Telegram / 企业微信 bot 推送。
- **图表展示**：K 线与财务图表（lightweight-charts / pyecharts / Plotly）。
- **界面**：默认用 Streamlit / Gradio 快速成型，避免独立前端工程；后续可选迁移到 FastAPI + 前端。

## 4. AI 分析模块需求（热插拔 · 本项目重点）

### 4.1 设计目标

AI 能力以**独立模块 + 适配器模式**实现，对核心系统完全可插拔：换提供商、换模型、换 Key、开关单个功能，均只改配置，不改业务代码；禁用时系统不报错、不阻塞。

### 4.2 统一接口契约（Provider-Agnostic Interface）

所有 AI 提供商实现同一抽象接口，输入输出**强制结构化（JSON）**，业务侧不感知具体模型：

```python
class AIAnalyzer(ABC):
    """所有 AI 功能的统一入口。实现类不得包含任何选股/回测的数值计算。"""

    @abstractmethod
    def analyze_sentiment(self, texts: list[str]) -> list[SentimentResult]:
        """接入点①：新闻/公告 → {sentiment: float[-1,1], confidence: float}"""

    @abstractmethod
    def extract_events(self, texts: list[str]) -> list[EventResult]:
        """接入点①：文本 → {ticker: str, event_type: str, confidence: float}"""

    @abstractmethod
    def nl_to_filter(self, instruction: str, field_schema: dict) -> FilterSpec:
        """接入点②：自然语言 → 受控的结构化筛选条件（DSL/参数化JSON，禁止生成可执行代码）"""

    @abstractmethod
    def generate_report(self, candidate_data: dict) -> str:
        """接入点③：基于传入的真实数据（RAG）综合成可读分析；禁止补充任何数字。"""
```

- **接入点①（数据层）**：情绪打分 + 事件抽取，输出落地为结构化因子列。**优先级最高**。
- **接入点②（引擎层边缘）**：仅生成筛选条件，**绝不执行**；执行由确定性引擎完成。须做输入校验，防注入。
- **接入点③（应用层）**：**必须 RAG**——只基于系统传入的真实/最新数据生成分析，prompt 中明确禁止模型自行补充数字，避免幻觉与知识过期。

### 4.3 配置驱动（API Key 与模型热切换）

通过配置文件 + 环境变量驱动，示例：

```yaml
ai:
  enabled: true                 # 全局开关；false 时整套系统仍可运行
  provider: anthropic           # anthropic | openai | local | none
  model: claude-haiku-4-5       # 初筛用便宜模型，精读可配贵模型
  api_key_env: AI_API_KEY       # Key 只从环境变量读取，禁止写入代码/仓库
  base_url: null                # 兼容自托管/兼容OpenAI协议的本地模型
  timeout_s: 30
  features:                     # 每个接入点可独立开关
    sentiment: true
    event_extraction: true
    nl_to_filter: false
    report_generation: true
```

- 用**工厂 + 适配器**：`get_analyzer(config)` 根据 `provider` 返回对应实现（`AnthropicAnalyzer` / `OpenAIAnalyzer` / `LocalAnalyzer`）。
- Key 仅从环境变量读取，**禁止硬编码或提交进版本库**。

### 4.4 优雅降级（强制行为）

- `enabled: false`、`provider: none`、或 Key 缺失时，工厂返回 `NullAnalyzer`：所有方法返回中性/空结果（情绪=0、无事件、报告为「AI 未启用」占位）。
- 此时确定性核心（指标 / 因子 / 筛选 / 回测）**必须完全正常**，不得抛错或中断。
- 单个 feature 关闭时，仅该接入点失效，其余不受影响。

### 4.5 进阶（可选）：Agent + 工具调用

若要给 AI 更大编排权限，采用 **LLM 编排 + 确定性工具执行**（function calling）：LLM 决定「查哪只、算什么、跑什么回测」，但所有数字由它调用的确定性工具函数返回。LLM 负责调度与推理，工具负责出数字——核心禁区原则不变。

### 4.6 成本控制

- 默认只对**自选池/候选池**（几十只）跑 AI，禁止默认对全市场逐条跑。
- 支持「便宜模型初筛 + 贵模型精读」分层。
- AI 调用须缓存（同文本不重复请求）。

## 5. 技术选型（默认轻量，均可替换）

| 层 | 默认 | 进阶 |
|---|---|---|
| 数据源（A 股） | akshare（主，免费）/ tushare（备，积分） | Wind / iFinD（付费） |
| 存储 | SQLite + Parquet | DuckDB / ClickHouse |
| 计算 | pandas + pandas-ta | polars |
| 回测 | backtrader / vectorbt | 自研事件驱动引擎 |
| 界面 | Streamlit / Gradio | FastAPI + 前端 |
| 调度 | cron / APScheduler | Airflow |
| AI | 兼容 OpenAI/Anthropic 协议，适配器化 | 本地模型（base_url 指向自托管） |

约束：能用现成库不造轮子，能用文件不上数据库，能用 Streamlit 不写前端。

## 6. 非功能需求与约束

- **可复现**：相同输入 + 相同配置，确定性核心输出必须完全一致。
- **无前视/无泄漏**：回测严禁未来函数；含 AI 特征的回测须警示并隔离前视污染风险（理想做法：标注 AI 特征的可信历史起点）。
- **数据正确性**：复权、停牌、幸存者偏差三项必须正确处理。
- **成本与速率**：AI 调用计费可控、带缓存与限流。
- **安全**：API Key 走环境变量；`nl_to_filter` 输出受控、防注入。
- **免责**：界面与报告须明确声明「本系统仅供研究，不构成投资建议；任何策略上实盘前须充分回测并计入成本，盈亏自负」。

## 7. 实施路线（分阶段，每阶段独立可用）

1. **MVP**：akshare 拉日线存 SQLite + Streamlit 跑基础条件筛选（PE / ROE / 均线）。
2. **指标与图表**：接 pandas-ta，画 K 线与财务图。
3. **回测**：用 backtrader 验证筛选条件历史有效性（含成本）。
4. **AI 模块骨架**：实现 `AIAnalyzer` 接口 + 工厂 + 配置 + `NullAnalyzer` 降级；先落地接入点①（舆情情绪因子）。
5. **因子与监控**：多因子打分排序 + 自选股预警推送。
6. **AI 报告/对话（接入点③）+ NL 筛选（接入点②）**。
7. **（可选）服务化**：迁移 FastAPI + 前端 + 调度。

---

## 8. 验收标准（Definition of Done）

- [ ] 确定性核心在 `ai.enabled=false` 下完整跑通选股 + 回测，无报错。
- [ ] 切换 `provider` / `model` / Key 仅改配置，业务代码零改动，AI 功能正常切换。
- [ ] 每个 AI feature 可独立开关；关闭后仅对应接入点失效。
- [ ] Key 缺失时自动降级到 `NullAnalyzer`，系统不崩。
- [ ] 核心代码中无任何对 LLM 的调用（可用静态检查/约定保证）。
- [ ] 回测正确处理复权、停牌、幸存者偏差，并计入交易成本。
- [ ] AI 报告基于传入真实数据生成（RAG），不含模型自行编造的数字。
- [ ] 界面含投资免责声明。

## 9. 增补需求（迭代）

> 在 MVP 之后新增的需求，记录于此以保持文档与实现一致。

### 9.1 荐股板块标注

选股结果须标注每只标的所属板块，便于区分交易规则（涨跌幅、开通门槛）与风格：

- 板块分类按证券代码前缀确定（确定性，无需外部数据）：
  - **主板**：沪 600/601/603/605、深 000/001/002/003（涨跌幅 ±10%）
  - **创业板**：深 300/301（注册制，±20%）
  - **科创板**：沪 688/689（注册制，±20%，50 万元开通门槛）
  - **北交所**：4/8 开头及 920（±30%）
- CLI 与 Web 候选列表均显示板块列；Web 端支持按板块过滤。

### 9.2 Web 专有名词悬浮解释

降低非专业用户理解门槛：界面上的专有名词（PE、ROE、夏普比率、最大回撤、后复权、
T+1、板块名等）右上角带 ⓘ 标识，**鼠标悬停即显示通俗解释**。

- 术语解释集中维护于 `aselect.glossary`（与 UI 解耦，可单测覆盖）。
- 表格列表头、关键指标、小节标题均挂载提示；不依赖第三方组件，用原生 tooltip 实现。
